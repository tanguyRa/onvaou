## Goal

Implement the Playwright-based scraper for official Mairie (Town Hall) agenda pages that do not expose a public API.

## Scope

### Scraper base class (`geo/harvesters/mairie_base.py`)

- Async Playwright context manager
- `random_sleep(min=2, max=8)` between page navigations
- Configurable `source_tag` per Mairie (e.g., `mairie_lyon`, `mairie_bordeaux`)
- Extracts: title, description, start date, location, address

### Concrete scrapers

Start with two reference implementations:

- `geo/harvesters/mairie_lyon.py` — targets `https://www.lyon.fr/agenda`
- `geo/harvesters/mairie_bordeaux.py` — targets `https://www.bordeaux.fr/agenda`

Each scraper must parse the page DOM and return a list of raw event dicts conforming to the internal `Event` model.

### Integration with normalizer

After scraping, pass raw events through the BAN normalizer and deduplication service before upsert.

### APScheduler job

`mairie_scrape_job` — daily at 04:00 CET, iterates all registered scrapers.

### Error handling

- Per-page errors must be caught and logged; the scraper must continue with remaining pages
- Failed scrapes are recorded in a `scrape_errors` log table (`scraper TEXT, url TEXT, error TEXT, ts DATETIME`)

## Acceptance Criteria

- [ ] Lyon and Bordeaux scrapers extract at least title + start_dt + address
- [ ] `random_sleep` is called between every page navigation
- [ ] Failed pages are logged to `scrape_errors` without crashing the job
- [ ] Scraped events pass through deduplication before upsert
