## Goal

Bootstrap the `geo/` Python service with FastAPI, project structure, Docker integration, and PostgreSQL + PostGIS database connectivity.

## Scope

### Directory structure

Create `geo/` at the workspace root with:

- `main.py` — FastAPI app entry point
- `config.py` — settings via `pydantic-settings` (env vars)
- `database.py` — asyncpg connection pool management
- `models/` — Pydantic response models
- `routers/` — FastAPI routers (events, health)
- `requirements.txt` — pinned dependencies
- `Dockerfile` — multi-stage build
- `.dockerignore`

### Key dependencies

`fastapi`, `uvicorn[standard]`, `pydantic-settings`, `asyncpg`, `geopandas`, `shapely`, `apscheduler`, `httpx`, `fuzzywuzzy`, `python-Levenshtein`, `playwright`

### Database connectivity

On startup, `database.py` must:

1. Create an `asyncpg` connection pool using `DATABASE_URL` (same Postgres instance as the Go backend)
2. Verify the `postgis` extension is enabled (`SELECT PostGIS_Version()`)
3. Expose a `get_pool()` dependency for FastAPI route injection
4. Gracefully close the pool on app shutdown (lifespan context manager)

### Health endpoint

`GET /health` → `{"status": "ok", "db": "connected"}`

### Docker integration

Add a `geo` service to `compose.yml`:

- Build from `./geo`
- Port `8000:8000`
- Environment: `DATABASE_URL` (points to the shared `postgres` service), `BAN_API_URL`, `OPENAGENDA_API_KEY`, `CORS_ALLOWED_ORIGINS`
- `depends_on: postgres: condition: service_healthy` (same healthcheck as the Go backend)

### `.env.example` additions

```
# Python Geo-Service
GEO_API_URL=http://onvaou-geo:8000
OPENAGENDA_API_KEY=
BAN_API_URL=https://api-adresse.data.gouv.fr
# DATABASE_URL is shared with the Go backend — no new variable needed
```

## Acceptance Criteria

- [ ] `docker compose up geo` starts without errors
- [ ] `GET /health` returns 200 with `{"status": "ok", "db": "connected", "postgis": "<version>"}`
- [ ] asyncpg pool connects to the shared Postgres instance
- [ ] Service is reachable from the `front` container via `GEO_API_URL`
