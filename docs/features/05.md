## Goal

Expose the core geospatial search API that the frontend consumes to display events on the map.

## Scope

### Endpoint

`GET /events`

**Query parameters:**


| Param       | Type   | Default | Notes                               |
| ----------- | ------ | ------- | ----------------------------------- |
| `city`      | string | —       | Mutually exclusive with `lat`/`lon` |
| `lat`       | float  | —       | WGS84 latitude                      |
| `lon`       | float  | —       | WGS84 longitude                     |
| `radius_km` | int    | 5       | 1–50, validated                     |
| `days`      | int    | 28      | 1–28, temporal window               |
| `page`      | int    | 1       | Pagination                          |
| `limit`     | int    | 50      | Max 200                             |


**Logic:**

1. If `city` provided → resolve via BAN API → `(lat, lon)`
2. Build a PostGIS query using `ST_DWithin(events.geom, ST_MakePoint(lon, lat)::geography, radius_meters)` — uses the GiST index automatically
3. Filter `start_dt BETWEEN now() AND now() + INTERVAL 'N days'`
4. Return paginated JSON

**Response model:**

```json
{
  "total": 142,
  "page": 1,
  "results": [
    {
      "event_id": "uuid",
      "title": "...",
      "start_dt": "2025-04-01T14:00:00Z",
      "address": "...",
      "lat": 45.75,
      "lon": 4.83,
      "source_tag": "openagenda",
      "source_url": "..."
    }
  ]
}
```

### Additional endpoints

- `GET /events/{event_id}` — full event detail
- `GET /cities/search?q=Lyon` — proxy to BAN API autocomplete (for frontend city picker)

### CORS

Allow origins from `CORS_ALLOWED_ORIGINS` env var (same pattern as Go backend).
