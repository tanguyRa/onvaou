## Goal

Extend the Go backend so authenticated users can save their preferred location and radius, and opt in to notifications about new nearby events.

## Scope

### PostgreSQL migration (`db/migrations/000006_user_location.up.sql`)

> Note: migration 000005 is reserved for the PostGIS events schema (see ticket:64b89706-c195-4bf3-9c66-670283a0d889/8f98713f-dd61-4caa-bd7b-2757bc5954ba).

Add to the `user` table:

- `saved_location GEOGRAPHY(Point, 4326)` — PostGIS geography point; replaces separate lat/lon columns for direct spatial join capability
- `saved_radius_km INTEGER DEFAULT 5`
- `notify_enabled BOOLEAN DEFAULT FALSE`
- `notify_push_token TEXT` — for future push delivery

Also add a GiST index on `saved_location` to support future "find all users near an event" queries:
`CREATE INDEX idx_user_saved_location ON "user" USING GIST(saved_location);`

### sqlc queries (`db/queries/auth_users.sql` additions)

- `UpdateUserLocation` — upsert lat/lon/radius
- `UpdateUserNotifyPrefs` — toggle `notify_enabled` + push token
- `GetUsersWithNotifications` — fetch users with `notify_enabled = TRUE` (for the notification worker)

### API routes (`back/internal/server/routes.go`)

- `PUT /api/secured/location` — save user location (requires auth)
- `PUT /api/secured/notifications` — update notification preferences (requires auth)
- `GET /api/secured/me` — return full user profile including location + notify prefs

### Notification worker (stub)

Create `back/internal/workers/notify.go` with a stub `NotifyNearbyEvents` function. It will query `GetUsersWithNotifications`, call the Python geo-service for each user's saved location, and dispatch push notifications. Full push delivery is out of scope for this ticket — the stub should log the intent.
