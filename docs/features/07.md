## Goal

Build the main event discovery UI: an interactive map with a city/ZIP search input, radius slider, and event cards.

## Scope

### New route: `/app/map` (`front/src/routes/(protected)/app/map/`)

This is the primary user-facing screen.

### Components

- `**MapView.svelte**` — Leaflet.js map (loaded via CDN to avoid SSR issues), renders event markers
- `**SearchBar.svelte**` — city/ZIP text input with autocomplete from `GET /cities/search`
- `**RadiusSlider.svelte**` — range input 1–50 km, debounced (300 ms) before triggering search
- `**EventCard.svelte**` — displays title, date, address, source badge; clicking opens detail panel
- `**EventDetailPanel.svelte**` — slide-in panel with full event description + source link

### Data fetching

- Call the Python geo-service via a SvelteKit server route `front/src/routes/api/events/+server.ts` (proxies to `GEO_API_URL` to avoid CORS from browser)
- Use Svelte 5 runes (`$state`, `$derived`, `$effect`) for reactive state

## Acceptance Criteria

- [ ] Map renders with Leaflet.js and shows event markers
- [ ] Typing in the search bar triggers city autocomplete from BAN API
- [ ] Moving the radius slider re-fetches events with the new radius (debounced)
- [ ] Clicking an event card highlights the marker and opens the detail panel
- [ ] Server route proxies correctly to `GEO_API_URL`
- [ ] Works on mobile (responsive layout: sidebar collapses below map)
