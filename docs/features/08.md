## Goal

Add a notification preferences section to the existing settings area so users can opt in to alerts about new events near their saved location.

## Scope

### New route: `/settings/notifications` (`front/src/routes/(protected)/settings/notifications/`)

### UI elements

- **Location picker** — city/ZIP input (same `SearchBar` component) + "Use my current location" button (Geolocation API)
- **Radius selector** — same `RadiusSlider` component
- **Notification toggle** — on/off switch for `notify_enabled`
- **Save button** — calls `PUT /api/secured/location` and `PUT /api/secured/notifications` on the Go backend

## Acceptance Criteria

- [ ] "Ma position" button triggers browser Geolocation API and pre-fills the city field
- [ ] Saving persists location + radius to Go backend (`PUT /api/secured/location`)

- [ ] Settings page is linked from the existing settings navigation
