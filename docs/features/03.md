## Goal

Implement the two primary data ingestion workers that fetch events from OpenAgenda and data.gouv.fr, normalize them, and upsert into PostgreSQL + PostGIS.

## Scope

### OpenAgenda harvester (`geo/harvesters/openagenda.py`)

- Paginated fetch from `https://api.openagenda.com/v2/events` using the API key
- Map response fields to the internal `Event` model
- Resolve location coordinates: use `geom` from OpenAgenda if present; otherwise call BAN API
- Set `source_tag = 'openagenda'`

### data.gouv.fr harvester (`geo/harvesters/datagouv.py`)

- Fetch relevant cultural/municipal datasets from `https://www.data.gouv.fr/api/1/datasets/`
- Parse CSV/JSON resources for event-like records
- Set `source_tag = 'datagouv'`

### BAN address normalizer (`geo/services/ban.py`)

- `async def resolve_address(address: str) -> tuple[float, float]`
- Calls `https://api-adresse.data.gouv.fr/search/?q=<address>&limit=1`
- Returns `(lon, lat)` in WGS84
- Implements retry with exponential backoff

### Deduplication (`geo/services/dedup.py`)

- Before upsert, compute a content hash of `(title, start_dt, source_tag)`
- Check against `source_hashes` table
- For near-duplicates across sources, use `fuzz.token_sort_ratio` (threshold ≥ 85) on `title + address`

### Upsert logic

- Use `INSERT INTO events ... ON CONFLICT (event_id) DO UPDATE SET ...` (PostgreSQL upsert)
- Update `source_hashes` after successful insert in the same transaction

### APScheduler jobs

Register in `geo/scheduler.py`:

- `openagenda_job` — daily at 03:00 CET
- `datagouv_job` — daily at 03:30 CET
- `vacuum_job` — weekly Sunday 04:00 CET (`VACUUM ANALYZE events; VACUUM ANALYZE source_hashes;`)

## Acceptance Criteria

Running openagenda_job manually populates the events tableDuplicate events (same title + date + source) are not inserted twiceNear-duplicates across sources are collapsed to a single recordBAN API is called for events missing coordinatesScheduler starts automatically with the FastAPI app lifespan
